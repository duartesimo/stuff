program starts
  main()
    read port + password
    create one Server object -> register handlers
    call server.run()

server setup
  open listening socket
  create poll loop state
  router already knows all command handlers
  add listener fd to poller with POLLIN

while (true)
  wait for poll events

  if no events
      restart loop

  if poll error
      if EINTR -> restart loop
      else stop loop

  for each fd returned by poll
      inspect its revents

      if listener readable
          accept raw socket
          get new client fd
          make client fd non-blocking
          build peer string from client address
          create Client
              Client contains IRC session state
              Client also contains Connection
                  Connection stores fd, peer, read buffer, write buffer
          add client to _clients map
              key = fd  ;   value = Client*
          add client fd to _poller with POLLIN

      else if client has error / hup / invalid fd
          close_client(fd)
              remove from channels
              remove from ops
              release nick
              remove from poller
              delete Client
              erase from _clients
              broadcast QUIT to channel members

      else if client readable
          find Client* from _clients using fd

          recv bytes from socket in a loop
              append bytes into this client’s read buffer
              keep reading until EAGAIN/EWOULDBLOCK
              if recv == 0 -> client disconnected -> close_client()
              if fatal recv error -> close_client()

          if client still exists
              extract complete lines from read buffer
                  Buffer checks for '\n'
                  incomplete trailing data stays in buffer
                  complete lines are returned as vector<string>

              for each complete line
                  call router.dispatch(fd, line)

                  dispatch does:
                      parse raw line into Message
                          raw
                          optional prefix
                          command
                          params
                      uppercase command
                      look up command in handler map
                      call matching handler -> handle_PASS / handle_NICK etc.
                      if no handler -> default handler

                  handler does command-specific logic
                      gets sender Client via fd
                      checks registration / params / permissions
                      reads or updates:
                          Client state
                          Channel state
                          server maps (_clients / _nicks / _chans)
                      may queue replies with srv.reply()
                      may finish registration -> PASS + NICK + USER -> queue welcome numerics + MOTD
                      may broadcast to channels
                      may disconnect client

      else if client writable
          find Client* from _clients using fd

          flush this client’s write buffer
              Connection::flush()
                  send queued bytes
                  erase bytes that were successfully sent
                  if socket would block -> keep remaining bytes for later
                  if fatal send error -> close_client()

          if write buffer is now empty
              update poller to watch only POLLIN
          else
              keep POLLOUT enabled so remaining queued output can be flushed later
